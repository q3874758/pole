# PoLE 隐私保护验证系统技术规格

**版本**：2.0

**设计理念**：无需登录、绝对隐私、本地验证

---

## 一、核心设计理念

### 1.1 问题陈述

传统游戏证明系统依赖中心化账号登录（如 Steam），存在以下问题：

- 隐私泄露：需要绑定真实身份
- 数据收集：平台收集大量用户行为数据
- 审查风险：账号可被封禁，资产可被冻结
- 体验割裂：需要额外登录步骤

### 1.2 解决方案

PoLE 2.0 采用**零知识证明 + 本地验证**架构：

- **无需登录**：生成加密钱包即可参与
- **绝对隐私**：所有数据留在本地，仅上传证明
- **行为证明**：验证游戏行为而非身份
- **防作弊**：硬件指纹 + 行为分析

## 三、核心技术实现

### 3.1 本地进程检测

**检测方法**：仅检测进程存在性，不收集任何数据

```go
// 伪代码示例 - 仅检测进程是否运行
func detectGameProcess() (bool, error) {
    // 获取正在运行的进程列表
    processes, err := process.Processes()
    if err != nil {
        return false, err
    }

    // 检查是否有已知游戏进程
    // ⚠️ 仅使用进程名进行模式匹配
    gameProcessNames := []string{
        "cs2.exe",           // CS2
        "Dota2.exe",         // Dota 2
        "Valorant.exe",      // Valorant
        "PUBG.exe",         // PUBG
        // ... 其他游戏
    }

    for _, p := range processes {
        name, _ := p.Name()
        for _, gameName := range gameProcessNames {
            if strings.Contains(strings.ToLower(name), gameName) {
                return true, nil  // 游戏运行中
            }
        }
    }

    return false, nil  // 无游戏运行
}
```

**隐私保护**：

- 仅检测进程是否存在
- 不读取进程内存
- 不读取进程网络连接
- 不读取窗口标题
- 不进行屏幕截图

### 3.2 零知识证明生成

**证明电路设计**：

```
输入（私密）：
- 运行时长 t
- 硬件指纹 H (已单向哈希)
- 进程签名 S

输出（公开）：
- 零知识证明 π

验证条件：
1. t > 最小时长阈值
2. S 是已知游戏的有效签名
3. H 符合格式（但不暴露原始硬件）
4. 人类操作特征存在（输入熵 > 阈值）
```

**使用的 ZK 技术**：

- **zk-SNARK**：轻量级，适合后台运行
- **Circom**：电路描述语言
- **Snarkjs**：证明生成库

**性能优化**：

- 证明生成时间：< 500ms
- CPU 占用：< 2%
- 内存占用：< 50MB
- 不影响游戏帧率

### 3.3 硬件指纹（隐私保护版）

**设计目标**：

- 检测同一台机器上的多个账户（防 Sybil 攻击）
- 不收集可识别个人的硬件信息

**实现方案**：

```go
// 单向哈希硬件指纹
func generateAnonymousHardwareID() string {
    // 收集硬件信息（不包含序列号）
    hwInfo := HardwareInfo{
        CPUName:  getCPUName(),           // 如 "Intel i7-12700K"
        CPUCores: getCPUCores(),          // 核心数
        GPUName:   getGPUName(),           // 如 "NVIDIA RTX 3080"
        RAMSize:   getRAMSize(),           // 内存大小
        OSName:    getOSName(),            // 操作系统
    }

    // 添加随机盐（每次安装生成，不同设备相同硬件也不同）
    salt := generateRandomSalt()

    // 组合并哈希
    data := fmt.Sprintf("%v-%s", hwInfo, salt)
    hash := sha256.Sum256([]byte(data))

    // 返回哈希值的前16位作为匿名ID
    return hex.EncodeToString(hash[:16])
}
```

**隐私保证**：

- 哈希过程不可逆
- 无法从哈希反推原始硬件
- 盐值本地保存，重装后生成新ID
- 不上传原始硬件信息

### 3.4 人类操作验证（输入熵）

**目的**：区分真实人类和自动化脚本

**检测方法**：

```go
// 计算输入熵
func calculateInputEntropy() float64 {
    // 记录键盘/鼠标输入
    // ⚠️ 所有计算在本地进行，不上传原始数据

    // 计算输入间隔的标准差
    // 人类操作：间隔随机，标准差大

    return stdDev
```

### 3.5 防虚拟机检测

**问题**：攻击者可能在虚拟机中运行游戏来获利

**解决方案**：

```go
// 检测虚拟机环境
func detectVirtualMachine() bool {
    indicators := 0

    // 1. 检查系统进程
    if containsVMProcesses() {
        indicators++
    }

    // 2. 检查硬件信息
    if hasVMHardwareFingerprint() {
        indicators++
    }

    // 3. 检查 CPU 特性
    if hasVMCPUID() {
        indicators++
    }

    // 4. 检查启动时间
    if hasSuspiciousBootTime() {
        indicators++
    }

    // 5. 检查内存大小
    if hasLowMemory() {
        indicators++
    }

    // 多项指标超标判定为虚拟机
    return indicators >= 3
}
```

### 4.2 差分隐私

**应用场景**：防止通过链上数据推断用户行为

```go
// 添加差分隐私噪声
func addDifferentialPrivacy(originalTime int64) int64 {
    // 拉普拉斯机制
    epsilon := 1.0  // 隐私预算
    scale := 3600   // 时间粒度（1小时）

    // 生成拉普拉斯噪声
    noise := laplaceNoise(epsilon, scale)

    // 添加噪声
    privateTime := originalTime + int64(noise)

    // 四舍五入到最近的30分钟
    privateTime = roundToNearest(privateTime, 1800)

    return privateTime
}
```

## 八、结论

PoLE 2.0 实现了以下核心价值：

1. **绝对隐私**：无需登录，不收集任何个人信息
2. **本地验证**：所有计算在用户设备完成
3. **零知识证明**：仅证明"玩了游戏"，不透露"玩什么游戏"
4. **防作弊**：多层次验证机制
5. **可持续**：玩游戏即获得奖励

**核心理念**：验证行为而非身份，证明而非披露。
